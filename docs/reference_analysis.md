# Wildcard Workshop: Reference Code Analysis (Revised)

본 문서는 `reference/` 폴더의 코드를 분석하여, 도구의 핵심 목표인 **"프롬프트 압축을 통한 효율적인 무작위 와일드카드 생성"** 로직을 정리한 문서입니다.

## 1. 핵심 목표
이 도구의 유일한 목표는 수많은 이미지 프롬프트 데이터를 **정제(Filtering)**하고 **압축(Compression)**하여, 고품질의 무작위 와일드카드 패턴을 만드는 것입니다. 

## 2. 핵심 알고리즘 및 단계

### 2.1 노이즈 제거 (Noise Reduction - Base Subtraction)
- **목표**: 모든 이미지에 공통적으로 나타나는 "당연한" 태그들을 제거하여 와일드카드의 변별력을 높입니다.
- **로직**: 기준(Base) 이미지를 하나 지정하면, 그 이미지에 포함된 모든 태그를 분석 대상 이미지들의 태그 세트에서 일괄 삭제합니다. 
- **효과**: 캐릭터 고유 특징이나 품질 태그(masterpiece 등)가 와일드카드에 포함되어 중복되는 것을 방지하고, 오직 "변화하는 부분"만 남깁니다.

### 2.2 태그 필터링 (Advanced Filtering)
- **목표**: 와일드카드 생성에 방해가 되는 기술적 태그나 원치 않는 키워드를 정밀하게 걸러냅니다.
- **Partial Match**: 단어의 일부만 포함되어도 삭제 (시스템 태그 등).
- **Exact Match**: 사용자가 Refiner를 통해 직접 눈으로 확인하며 빈도수가 높거나 불필요하다고 판단한 태그를 정확히 일치시켜 삭제.
- **Exceptions**: 필터링 규칙에 의해 실수로 삭제되면 안 되는 중요한 단어를 보호.

### 2.3 유사도 기반 압축 (Compression - Recursive Merging)
- **목표**: 내용이 거의 비슷한 여러 개의 프롬프트를 하나로 합쳐 전체 데이터의 부피를 줄입니다.
- **로직**: 
    1. Jaccard Similarity를 이용해 태그 구성이 유사한 이미지 그룹을 찾습니다.
    2. 그룹 내에서 **공통된 부분(Intersection)**을 밖으로 빼고, **차이가 나는 부분(Difference)**을 `{option1|option2}` 형태로 묶습니다.
    3. 이 과정을 재귀적으로 반복하여 매우 복잡한 프롬프트 세트도 `base{diff1{a|b}|diff2}`와 같이 계층적으로 압축된 한 줄의 와일드카드로 변환합니다.

## 3. 표준 워크플로우 (Expected Workflow)

### 단계 1: 데이터 로드 (Import)
- 와일드카드로 만들 이미지 폴더나 개별 파일들을 로드합니다.

### 단계 2: 베이스 클리닝 (Base Cleaning - 선택 사항)
- 공통적으로 들어있는 '기본값'을 제거하기 위해 기준 이미지를 설정합니다. 이는 와일드카드를 훨씬 가볍고 핵심적으로 만들어줍니다.

### 단계 3: 인터랙티브 정제 (Refinement)
- Refiner를 열어 전체 태그의 빈도수를 확인합니다. 
- 와일드카드 생성 시 노이즈가 될 수 있는 태그들을 체크하여 필터에 등록합니다.

### 단계 4: 압축 실행 (Compression)
- 유사도 임계값(Threshold)을 조절하여 압축 강도를 결정합니다.
- 'Run Workshop'을 통해 최종적으로 정제되고 압축된 와일드카드 리스트를 얻습니다.

### 단계 5: 결과 활용
- 생성된 와일드카드를 복사하여 설정 파일에 저장하거나 ComfyUI 등에서 무작위 프롬프트 생성용으로 사용합니다.
