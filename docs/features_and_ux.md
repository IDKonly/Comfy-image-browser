# 기능 명세 및 사용자 경험 (Features & UX)

ComfyView의 모든 기능은 실제 AI 아트 생성자들의 워크플로우를 관찰하고, 그들의 불편함을 해결하기 위해 설계되었습니다. 이 문서는 주요 기능의 구현 배경과 UX 설계 철학을 설명합니다.

## 1. 배치 크롭 시스템 (Batch Crop System)

LoRA 학습이나 썸네일 제작을 위해 수십 장의 이미지를 같은 비율과 위치로 잘라내는 과정은 매우 고된 작업입니다.

*   **배경**: 기존 도구들은 이미지 한 장씩 크롭하거나, 복잡한 파이썬 스크립트를 작성해야 했습니다.
*   **ComfyView의 해결책**: 시각적인 캔버스 위에서 마우스로 영역을 지정하면, 같은 폴더 내의 모든 이미지에 대해 동일한 좌표로 크롭 작업을 수행합니다. `image-rs`를 활용한 고성능 처리를 통해 대량의 이미지를 즉시 처리할 수 있습니다.
*   **UX 설계**: 실시간 프리뷰를 제공하여 작업 실수를 방지하고, 단일 버튼 클릭으로 전체 배치 작업을 완료할 수 있도록 설계되었습니다.

---

## 2. 파일 분류 및 실행 취소 (Sorting & Undo System)

이미지를 분류하는 과정은 '결정'의 연속입니다. 사용자가 잘못된 결정을 하더라도 시스템은 이를 안전하게 보호해야 합니다.

*   **Keep & Trash**: 이미지를 영구 삭제하는 대신 `_keep`과 `_trash` 폴더로 이동시키는 비파괴적 분류 방식을 채택했습니다. 이는 실수를 방지하고, 나중에 필요할 경우 폴더를 다시 스캔하여 이미지를 복구할 수 있게 합니다.
*   **Ctrl+Z (Undo)**: `useAppStore.ts` 내에 `UndoAction` 스택을 구현하여 이미지 이동 작업을 언제든지 되돌릴 수 있습니다. 사용자는 마음 편히 이미지를 분류하고, 실수했을 때 즉시 원래 상태로 되돌릴 수 있습니다.

---

## 3. 와일드카드 워크숍 (Wildcard Workshop)

창작자들은 수백 번의 생성 과정(Batch)을 통해 최적의 프롬프트를 찾습니다.

*   **배경**: 생성된 수많은 이미지 중 어떤 프롬프트 조각이 좋은 결과를 냈는지 찾는 것은 어렵습니다.
*   **기능 구현**: 메타데이터를 기반으로 프롬프트의 공통점과 차이점을 분석하여 '와일드카드' 형태로 요약해 줍니다. Jaccard 유사도 분석을 통해 가장 가치 있는 프롬프트 조합을 우선적으로 추천합니다.
*   **확장성**: `MergeFilter` 기능을 통해 여러 개의 프롬프트 파일이나 클립보드 데이터를 하나로 합쳐서 새로운 생성 실험을 위한 와일드카드를 만들 수 있습니다.

---

## 4. 시각적 경험 및 접근성 (Visual Experience)

*   **Zoom & Pan**: 고해상도 이미지를 세밀하게 관찰할 수 있도록 리액트의 고성능 트랜스폼 기술을 활용한 줌/팬 기능을 제공합니다.
*   **Keyboard First**: 마우스 없이도 이미지 브라우징, Keep/Trash 분류, 검색 창 활성화 등이 가능하도록 풍부한 단축키 시스템을 갖추고 있습니다.
*   **Toast System**: 백그라운드에서 일어나는 모든 파일 작업의 성공/실패 여부를 사용자에게 즉각적이고 비침습적으로 알립니다.
