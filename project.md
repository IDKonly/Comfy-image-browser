# Batch Crop (다중 영역 일괄 자르기) 기능 통합 계획 및 요구사항

## 1. 프로젝트 목적

이 기능의 핵심 목적은 **"하나의 이미지에서 한 개 이상의 원하는 영역(Region)을 선택하여 한 번에 일괄적으로 자르는 것(Batch Crop)"**입니다. 기존의 단일 영역 자르기의 한계를 넘어, 반복적이고 세밀한 이미지 분할 작업을 직관적이고 빠르게 처리할 수 있도록 돕습니다.

## 2. 기능적 요구사항 (Functional Requirements)

### 영역 규격 복사 및 붙여넣기 (Copy & Paste)

사용자가 현재 선택한 사각형 영역의 규격(가로, 세로 크기)을 단축키(예: Ctrl+C / Ctrl+V)를 통해 복사 및 붙여넣기 할 수 있어야 합니다.
이를 통해 동일한 크기의 자르기 영역을 이미지 내 여러 곳에 빠르게 배치할 수 있습니다.

### 드래그 이동 및 스냅(Snap) 가이드 기능

캔버스 위에 생성된 개별 사각형 영역은 마우스로 클릭하여 자유롭게 드래그 및 이동이 가능해야 합니다.
사각형을 이동하거나 크기를 조절할 때, 다른 사각형의 모서리나 이미지의 가장자리에 근접하면 자석처럼 달라붙는 스냅(Snap) 및 가이드라인 기능이 제공되어야 합니다. (정밀한 위치 조정 지원)
이 기능은 사용자의 필요에 따라 툴바 등의 UI 옵션을 통해 스냅을 켜고 끄거나(Toggle), 슬라이더를 통해 자력의 민감도(Sensitivity, 예: 5px~30px 픽셀 범위)를 직관적으로 조절할 수 있어야 합니다.
특정 키(예: Alt 키)를 누른 상태로 드래그하거나 스냅을 의도적으로 무시하려는 빠른 마우스 조작이 감지될 경우, 스냅 기능을 잠시 동안 비활성화하여 부드럽고 자유로운 이동을 보장합니다.

### 독립된 수정 윈도우 (New Window/Overlay Mode)

메인 프로그램 창에서 특정 이미지의 '편집(또는 Batch Crop)' 버튼을 누르면, 메인 UI와 분리된 **새로운 수정 윈도우(또는 전체 화면 오버레이)**가 열려야 합니다.
모든 크롭 편집 작업은 이 새로운 윈도우 내에서 독립적으로 처리되며, 완료(저장) 또는 취소 시 창이 닫히고 메인 창으로 돌아갑니다. 단, 창을 닫거나 취소할 때 미저장된 슬라이스 영역이 있다면 사용자에게 경고(Confirm) 창을 표시하여 데이터 유실을 방지합니다.

### 일관된 디자인 언어 및 이미지 중심 레이아웃

편집 창의 레이아웃은 기존 Comfy-image-browser의 디자인 원칙(어두운 테마, 모던 UI)을 그대로 계승해야 합니다.

방해 요소를 최소화하고 화면의 대부분을 이미지가 차지하는 이미지 중심(Image-centric) 레이아웃을 지향하여, 작업 몰입도를 높입니다.

### 독립된 모듈 및 코드 구조 (Independent Module)

이 Batch Crop 기능은 기존 코드의 상태(Store)나 로직을 오염시키지 않는 **완전히 독립된 모듈(컴포넌트 및 Rust 백엔드)**로 작성되어야 합니다.
메인 앱에서는 단지 이 모듈을 호출하고 필요한 매개변수(이미지 경로 등)만 전달하면 작동하는 플러그인(Plug-in)과 같은 형태로 설계됩니다.

## 3. 아키텍처 및 코어 구성 요소 (Architecture)

### 3.1. Frontend 독립 모듈 (BatchCropModule.tsx)

- 역할: 사용자가 이미지를 보고 여러 개의 자르기 영역을 그리는 팝업/새 창 형태의 UI.
- 독립성 보장: 모듈 내부에서 자체적인 상태 관리(useState, useRef)를 사용하며, 전역 상태(Zustand 등)에 의존하지 않습니다.
- 주요 기능 컴포넌트:
    - CropCanvas: 이미지 렌더링 및 드래그, 스냅 로직 처리 영역.
    - SliceOverlay: 개별 크롭 영역(사각형). 크기 조절, 선택, 이동을 담당.
    - Toolbar: 복사/붙여넣기 단축키 힌트, 스냅 활성화 토글 및 민감도 슬라이더, 저장, 닫기 버튼 등을 포함하는 상단/하단 컨트롤 바.

### 3.2. Backend 모듈 (src-tauri/src/crop.rs)

- 역할: 프론트엔드에서 전달받은 원본 이미지 경로와 크롭 좌표 리스트(x, y, width, height)를 바탕으로 실제 이미지를 자르고 디스크에 저장합니다.
- 의존성: Rust의 image 크레이트 사용.
- 명령어: #[tauri::command] process_batch_crop과 같은 전용 함수로 분리하여 메인 로직과 충돌하지 않도록 구현합니다. 성공 시 저장된 파일들의 경로 목록을 반환하고, 실패 시 명확한 에러 메시지(Result<Vec, String>)를 프론트엔드로 전달하여 적절한 Toast 알림을 띄울 수 있도록 설계합니다.

## 4. 구현 단계 (Roadmap)

### Phase 1: 기반 아키텍처 및 프론트엔드 모듈 설계

메인 창에서 진입할 수 있는 트리거(버튼) 생성.
전체 화면 오버레이 또는 Tauri 새 창 형태로 BatchCropModule 레이아웃 구성.

### Phase 2: 인터랙티브 캔버스 구현 (스냅 및 복사/붙여넣기)

이미지 로드 및 마우스 이벤트(드래그 앤 드롭) 연동.
Ctrl+C / Ctrl+V 이벤트 리스너를 통한 크기 복제 로직 구현.
X, Y 축 기준 스냅(Snap) 계산 로직 적용.

### Phase 3: 백엔드 크롭 파이프라인 (Rust)

crop.rs 모듈 생성 및 다중 이미지 분할 저장 로직 구현.
비동기 처리 및 프론트엔드와의 통신(IPC) 연결.

### Phase 4: 디자인 다듬기 및 테스트

- 기존 프로그램의 테마(색상, 아이콘 등)와 맞추기 위한 CSS/Tailwind 클래스 미세 조정.
- 대용량 고해상도 이미지 처리 성능 테스트 및 엣지 케이스 예외 처리 (예: 지정한 자르기 영역이 이미지 원본 크기를 벗어나는 경우 사용자가 설정한 색으로 벗어난 영역을 자동으로 채웁니다. 기본값은 흰색입니다., 디스크 용량 부족, 잘못된 파일 경로 입력 등).
