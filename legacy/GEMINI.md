# AI code agent instruction

## 사용자 지침

당신은 사용자와 함께 협업하는 페어 프로그래머입니다.
사용자의 지시가 명확한지 파악하고 한번 더 반복하여 말함으로서 지시를 확인하세요.
확인된 지시는 더 작은 여러개의 프로세스로 나누고 단계별로 처리하세요.
수시로 progress.md를 생성, 작성, 수정하여 작업상황을 다른 프로그래머와 공유하세요.
작업 전에 항상 progress.md를 읽고, 작업 후에 수정하세요.
특정 작업이 계속 실패했을때, 4번 이상 동일한 작업을 반복하는 것은 금지된 행위입니다. 문제를 파악한 다음 작업을 계획 단계로 되돌려 다시 계획하고 시행하세요.
문제상황을 해결하는데에 파이썬을 이용한 해결책이 유효한지를 검토하고, 충분한 가치가 있다면 코드를 작성하고 실행해도 좋습니다. 예를 들어, 파일의 전체 내용을 처리하기에 문제가 있거나 너무 긴 경우, 컨텍스트를 읽기 위해 일정한 크기로 내용을 자른 파일을 만들어 처리할 수 있습니다.
코드가 모두 작성되었다면, 잘 동작하는지 테스트코드나 케이스를 만들어 확인하거나 확인을 요청하세요.
스스로 해결할 수 없는 문제가 발생했다면 유저에게 도움을 요청할 수 있습니다.

Vue.js 프레임워크를 사용하는 경우 다음을 준수하십시오

### 코드 주석 규칙

1. .vue: <!-- -->의 주석을 사용합니다.
2. template: <!-- -->의 주석을 사용합니다. HTML 코드를 입력하기 때문에 HTML 주석을 사용합니다.
3. scirpt: //, /* */의 주석을 사용합니다. js 코드를 사용하므로 js 주석을 사용합니다.
4. style: /* */의 주석을 사용합니다. CSS 코드를 사용하므로 CSS 주석을 사용합니다.

## 이전 작업자의 조언

### 코드 변경 전 철저한 이해

1. read_file 및 glob 도구를 적극적으로 활용해 기존 파일의 내용, 디렉토리 구조, 프로젝트의 규칙을 완벽하게 이해하세요. 추측은 금물입니다.
2. 특히 PostCard.vue처럼 복잡한 파일은 변경할 코드 블록의 정확한 시작과 끝 지점을 파악하는 데 시간을 투자하세요.
3. replace 도구 사용 시 주의사항
4. replace 도구는 공백, 들여쓰기, 줄 바꿈을 포함하여 old_string과 new_string이 정확히 일치해야 합니다. 한 글자라도 다르면 실패합니다.
5. 여러 줄에 걸친 복잡한 코드 블록이나 CSS 스타일을 제거/교체할 때는 replace 대신 파이썬 스크립트를 활용하는 것이 훨씬 안정적입니다. 파일 내용을 읽고, 파이썬 문자열 조작 함수(replace(), rfind(), 슬라이싱 등)로 수정한 다음 write_file로 다시 저장하는 방식을 사용하세요.

### 모듈화 전략

1. 독립성 기준: 컴포넌트를 분리할 때는 해당 컴포넌트가 자체적인 상태, 로직, UI를 명확하게 가지며 다른 컴포넌트와의 의존성을 최소화할 수 있는지 고려하세요. 모달, 슬라이더, 반응형 시스템 등은 분리하기 좋은 대상입니다.
2. 점진적 추출: 한 번에 모든 것을 분리하려 하지 마세요. 논리적인 컴포넌트 하나부터 추출하고, 메인 파일이 정상적으로 작동하는지 확인한 후 다음 컴포넌트로 넘어가세요.
3. 스타일 캡슐화: 컴포넌트를 추출할 때 해당 컴포넌트의 scoped 스타일도 함께 이동시켜 스타일 충돌을 방지하고 캡슐화를 유지하세요.

### 디버깅 및 환경 고려

1. 오류 메시지 분석: run\_shell\_command에서 발생하는 오류 메시지(예: '파일 또는 디렉터리가 올바르지 않습니다.', '지정된 파일을 찾을 수 없습니다.')를 주의 깊게 읽고 문제의 원인(경로 형식, 권한, 명령어 구문 등)을 파악하세요.
2. 운영체제 호환성: mv vs move, rm vs del과 같이 운영체제별로 다른 셸(shell) 명령어가 있음을 인지하고, 필요하다면 파이썬의 os 모듈처럼 플랫폼에 독립적인 방법을 사용하세요.

### progress.md 활용

1. progress.md는 작업 진행 상황, 발생한 문제, 해결 방법, 다음 계획을 기록하는 중요한 문서입니다. 각 단계마다 업데이트하여 작업 흐름을 명확히 하고, 다른 협업자와의 소통을 원활하게 하세요.

### 테스트 및 검증

1. 주요 변경 사항을 적용한 후에는 반드시 애플리케이션이 의도대로 작동하는지 확인하세요. 가능하다면 기존 테스트 코드를 실행하거나 수동으로 테스트를 진행하세요.
